# Code Changes Policy

Use this policy for code edits in any repository unless a project-specific policy overrides it.

- Inspect the target file(s) before proposing or applying edits.
- Prefer editing existing files over creating new files when both options satisfy the task.
- Keep diffs small and focused to the requested outcome.
- Avoid premature abstractions; introduce shared helpers only when duplication is meaningful.
- Add error handling, comments, or scaffolding only when required for correctness, safety, or maintainability of the requested behavior.
